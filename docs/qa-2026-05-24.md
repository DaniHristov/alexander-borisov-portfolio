## Accessibility (static inspection)

- **skip-link first in body** — PASS. `layout.tsx` line 34: `<a className="skip-link" href="#main">` is the first child inside `<body>`, before `<Nav />`.
- **`<html lang="en">`** — PASS. `layout.tsx` line 32: `<html lang="en" className={inter.variable}>`.
- **Single `<h1>` per page** — PARTIAL PASS with one concern:
  - `/` (home) — PASS: one `<h1>` inside `Hero` component (`Hero.tsx` line 6).
  - `/about` — PASS: one `<h1>` in `src/app/about/page.tsx` line 12.
  - `/contact` — PASS: one `<h1>` in `src/app/contact/page.tsx` line 11.
  - `/works/_example-poster` (project detail) — PASS: one `<h1>` in `ProjectMeta.tsx` line 10.
  - `/not-found` — PASS: one `<h1>` ("Not found") in `not-found.tsx` line 7.
  - **`/works` — CONCERN: NO `<h1>` present**. The works listing page (`src/app/works/page.tsx`) renders `TagFilter`, `ViewToggle`, `WorksGrid`/`IndexList` but no heading element. Recommend adding a visually-hidden `<h1>Works</h1>` (e.g. `sr-only` class) to satisfy WCAG 1.3.1 (Info and Relationships) and give screen-reader users a page landmark.
- **All `<Image>` have `alt`** — PASS. All three `<Image>` usages supply `alt` from data:
  - `ProjectCard.tsx` line 20: `alt={project.cover.alt}`
  - `ProjectGallery.tsx` line 22: `alt={img.alt}`
  - `about/page.tsx` line 20: `alt={site.about.portrait.alt}`
- **All `target="_blank"` have `rel="noreferrer noopener"`** — PASS. All three instances confirmed:
  - `Footer.tsx` line 16–17: `target="_blank" rel="noreferrer noopener"`
  - `contact/page.tsx` line 31–32: `target="_blank" rel="noreferrer noopener"`
  - `about/page.tsx` line 59: `target="_blank" rel="noreferrer noopener"`
- **`min-h-[44px]` on interactive links in Nav, Footer, TagFilter, ViewToggle, Contact** — PASS. All interactive link elements confirmed:
  - Nav: logo link (line 18), nav links (line 27)
  - Footer: social links (line 18)
  - Contact: mailto link (line 21), social links (line 33)
  - TagFilter: "All" link (line 35), tag links (line 49)
  - ViewToggle: grid link (line 31), index link (line 41)
- **`aria-current` used on filter/toggle links (not `aria-pressed`)** — PASS. `aria-pressed` not found anywhere. `aria-current="page"` used correctly:
  - `TagFilter.tsx` lines 34 and 48: `aria-current={active === null ? 'page' : undefined}` / `aria-current={active === t ? 'page' : undefined}`
  - `ViewToggle.tsx` lines 30 and 40: `aria-current={!isIndex ? 'page' : undefined}` / `aria-current={isIndex ? 'page' : undefined}`
